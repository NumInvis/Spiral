中间过程优化说明

  1. 自由文本画像解析

    由 LLM 从自然语言中提取省份、科类、位次、意向城市与专业；缺少关键信息时直接报错，不预设默认值。系统默认考生服从专业组内调剂。

  2. 专业组代码与组内专业说明

    每一行推荐中的专业组代码（如 A16707）对应院校的一个专业组。代码后的描述（如"含电气工程、自动化等方向"）直接来自该组内最相关的专业名称，帮助用户快速判断这个组是什么项目。详情可展开查看组内全部专业。

  3. 官方投档线为准

    2025 年推荐完全基于湖北省教育考试院官方 PDF 投档线表与一分一段表；2024/2023/2022 年数据来自开源整理的官方投档/录取数据。不自行估算专业热度或捏造专业线。

  4. 冲稳保分档

    以考生位次与官方院校专业组投档位次的差距估算录取概率，结合考生风险偏好划分冲/稳/保。保档上限不超过考生位次×130%，避免志愿过度浪费。

  5. 智能排序（数据筛选 + 层次加权 + LLM 决策）

    同档次内先按"学校层次（985/211/双一流）+ 专业匹配度 + 录取概率"综合排序。若配置了 LLM API key，则由 LLM 对候选池进行最终排序，生成自然语言推荐理由和风险提示；未配置时回退到规则排序。

  6. 数据置信度

    2024 年真实专业录取线标 A，专业组投档线标 B；2025 年仅公布院校专业组投档线，统一标 C。所有置信标签随原始数据传播到推荐结果。
