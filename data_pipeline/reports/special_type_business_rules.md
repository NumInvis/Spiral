> 版本：2025-07-11  
> 适用范围：湖北省（物理类/历史类）本科普通批

## 2. 用户意图检测规则（Explicit Consent）

### 2.1 开启条件（任一满足即视为用户明确接受）

用户自由文本描述中必须包含：

**正向触发词**（接受/愿意/可以/填报/报/考虑/走）+ **0~5 个字符** + **特殊类型关键词**

覆盖的关键词包括：
- 国家专项、地方专项、高校专项
- 预科、定向、民族班
- 援藏、南疆、边防军人子女
- **中外合作、护理类、高收费、公费师范、国际班、马来西亚分校**

### 2.2 否定排除（优先于正向触发）

如果用户文本中包含：
**否定词**（不要/不想/不填/不接受/拒绝/排除）+ **0~5 个字符** + **特殊类型关键词**

则 `allow_special_types = False`，即使同时出现正向触发词。

### 2.3 默认行为

- **未提及任何特殊类型关键词** → `allow_special_types = False`
- **仅提及但未表达接受意愿**（如"听说有国家专项"）→ `allow_special_types = False`
- LLM 解析结果优先：如果 LLM 明确识别出 `allow_special_types` 字段，则使用 LLM 结果；否则回退到规则引擎。

## 3. 推荐引擎隔离逻辑

### 3.1 数据流隔离

```
CSV 原始数据 (18,430 行)
    │
    ├─► 普通类记录 (~14,225 行, 77.2%)
    │     └─► 按院校+专业组代码聚合
    │     └─► 按冲/稳/保分档 (10/25/10)
    │     └─► 输出 recommendations[]
    │
    └─► 特殊类型记录 (~4,205 行, 22.8%)
          ├─► 用户 allow_special_types=False → 全部过滤丢弃
          └─► 用户 allow_special_types=True → 进入特殊类型处理流
                └─► 按院校+专业组代码聚合
                └─► 按冲/稳/保分档 (3/8/3，数量更少)
                └─► 输出 special_recommendations[]
                └─► 按类型名称分组 → special_by_type{}
```

### 3.2 冲稳保分档策略（特殊类型独立计算）

特殊类型使用与普通类相同的概率区间，但候选数量上限不同：

| 风险偏好 | 冲区间 | 稳区间 | 保区间 | 普通类目标数 | 特殊类目标数 |
|---------|--------|--------|--------|-------------|-------------|
| aggressive | 5%~30% | 30%~60% | ≥60% | 10/25/10 | 3/8/3 |
| balanced | 15%~45% | 45%~75% | ≥75% | 10/25/10 | 3/8/3 |
| conservative | 20%~45% | 55%~85% | ≥85% | 10/25/10 | 3/8/3 |

### 3.3 数据结构

`build_recommendation()` 返回字典包含：

```python
{
    "profile": Profile,
    "total_groups": int,          # 普通类推荐组数
    "冲_count": int,
    "稳_count": int,
    "保_count": int,
    "recommendations": List[RecommendationItem],        # 普通类
    "special_recommendations": List[RecommendationItem],  # 特殊类（仅当 allow_special=True）
    "special_by_type": Dict[str, List[Dict]],          # 按类型分组，如 {"国家专项计划": [...], "中外合作办学": [...]}
    "warnings": List[str],        # 若 allow_special=False 且存在特殊类记录，提示用户
}
```

## 4. 报告展示规则

### 4.1 普通类推荐表格

- 始终展示，包含所有冲/稳/保专业组
- 列：序号、分档、院校、专业组代码、2024最低位次、2025最低位次、录取概率、详情

### 4.2 特殊类型推荐表格（仅当 `allow_special_types=True`）

- 在普通类表格之后、方法论附录之前独立展示
- 标题：**特殊类型招生 · 单独列出 · 仅在用户明确要求时展示**
- 顶部附加说明：提醒用户特殊类型有额外报考条件或培养模式限制
- 按类型分小节展示，每小节带类型名称（如"国家专项计划"、"中外合作办学"）
- 列：序号、分档、院校、专业组、概率、详情
- 不展示年份位次列（特殊类型历年数据更不稳定）

### 4.3 警告提示（当 `allow_special_types=False` 时）

如果系统检测到用户位次附近存在特殊类型招生计划，但用户未明确要求，则返回警告：

> "检测到特殊类型招生计划，但用户未明确要求。如需查看，请在描述中提及"国家专项/中外合作/预科"等关键词。"

## 5. 数据清洗规则（CSV 原始数据）

### 5.1 字段映射与校验

| CSV 字段 | 系统映射 | 校验规则 | 备注 |
|---------|---------|---------|------|
| 院校名称 | School.name | 非空，去重 | — |
| 院校代码 | School.code | 4位数字字符串 | 不足4位前补0 |
| 专业组代码 | Major.group_code | 2位字符串 | 如"01"、"02" |
| 专业名称 | Major.name | 非空 | 含特殊类型关键词的进入特殊流 |
| 专业组备注_2025 | Major.description | 可为空 | 2025年批次类型标注来源 |
| 最低分/最低分.1 | — | **不视为真实专业线** | 38%低于组线，不可信 |
| 录取最低分 | — | **100%为空** | 无真实专业录取线数据 |
| 最低位次 | MajorScore.group_lowest_rank | 非空时可信 | 2024年经官方交叉验证 |

### 5.2 数据可信度标注

| 年份 | 数据类型 | 置信度 | 来源 |
|------|---------|--------|------|
| 2024 | 院校专业组投档线 | **B** | 官方PDF交叉验证 (95%+匹配) |
| 2025 | 院校专业组投档线 | **C** | 模拟演练/预估数据，非正式录取 |
| 2022-2023 | 院校专业组投档线 | **B** | 开源整理 |
| 所有年份 | 真实专业录取线 | **A**（理论上） | **数据缺失，不可用** |

> **关键约束**：系统明确声明不估算专业热度，不做"专业线 = 组线 + 热度调整"的虚构计算。所有推荐概率仅基于官方组线位次与考生位次的比值。

### 5.3 异常值检测规则

- **组-专业线倒挂**：若某专业 `最低分` < `组线最低分` 超过 5 分，标记为数据异常（可能是不同批次或不同统计口径）
- **同一院校组内位次差过大**：同一组内不同专业 `最低分` 差距 > 100 分，标记为异常（可能是合并了多个方向）
- **学费异常**：学费 > 10,000 元/年 且含"中外合作/国际/高收费"关键词，标记为高收费类；学费 > 50,000 元/年 标记为极高收费（如中外合作医学类）

## 6. 关键系统约束与免责声明

1. **专业线数据缺失**：系统没有真实专业录取线（A级数据），所有概率估算基于专业组投档线，对热门专业的难度系统性低估。
2. **2025年数据性质**：`LATEST_HISTORICAL_YEAR=2025` 的数据主要是模拟演练/预估数据，非正式录取结果，使用时需明确标注为 C 级置信度。
3. **特殊类型资格限制**：系统仅做数据层面的类型识别和分类，**不验证用户是否符合特殊类型报考条件**（如国家专项的户籍要求、民族班的民族身份要求、公费师范的签约要求等）。用户必须自行确认资格。
4. **概率公式为数学构造**：`estimate_probability()` 的公式基于位次比值，无历史录取统计验证，仅供相对排序参考，不视为真实录取概率。

## 8. 后续待优化项

- [ ] **高收费过滤**：用户未要求中外合作时，高收费（>10,000/年）的普通专业组是否也应归入特殊类型？当前按名称关键词归类，可能遗漏部分高收费普通专业。
- [ ] **提前批区分**：当前 CSV 中所有记录 `批次` 字段均为"本科批"，无"提前批"区分。若后续数据源补充提前批数据，需在 `classify_special_type()` 中增加批次字段判断。
- [ ] **专业线数据补全**：若能获取真实专业录取线（A级数据），可重构概率估算，实现专业级精准推荐，而非仅组级推荐。
- [ ] **多省扩展**：当前 `province_rules/hubei.json` 仅定义湖北省规则，其他省份需单独定义特殊类型关键词和批次规则。
